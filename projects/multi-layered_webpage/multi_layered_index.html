Multi-Layered Webpage

Project Overview

The Project Registration System is a comprehensive web-based application that allows users to register, browse products, showcase portfolios, and participate in polling. Designed with user experience in mind, this application provides a seamless interaction for various functionalities including e-commerce, personal displays, and community engagement.

Project Structure

  1. Base Page (base.html)
    Purpose: The Base Page serves as the foundational template for the Project Registration system. It provides a consistent layout and styling for all subsequent pages, ensuring a cohesive user experience. Key elements like headers, footers, and navigation links are included to guide users through the application.

Project Registration Page (index.html)

This page allows users to register by inputting personal details, including name, contact information, and demographic data. The user-friendly form facilitates easy onboarding, making the registration process straightforward and efficient.

E-Commerce Page (e-commerce.html)

The E-Commerce Page showcases various products, enabling users to browse items, view product descriptions, and add them to a shopping cart. This interactive platform enhances the shopping experience by making it engaging and user-centric.

Portfolio Page (personal_portfolio.html)

The Portfolio Page presents a user's personal information, work experience, education, and skills. It acts as a digital resume, allowing users to effectively showcase their qualifications and professional journey.

Polling Selection Page (polling_selection.html)

This page features a polling system that enables users to vote for their favorite images (e.g., dogs or cats) while collecting demographic information. It promotes user engagement and collects valuable data for further analysis.

Form Data (formdata.json)

The formdata.json file is used to store submitted user data in a structured format. This ensures that the information is persistently saved and easily retrievable for analysis and further processing.

Server File (server.js)

The Server File sets up a local server using Express, handling incoming form submissions and saving the data to formdata.json. It enables the backend functionality required for the application, ensuring smooth data flow between the client and server.

Technologies Used

Frontend Technologies

  • HTML5: Structured the content across all pages, ensuring semantic and accessible markup.
  • CSS3: Enhanced visual appeal and provided responsive design, adapting to different screen sizes.
  • JavaScript: Implemented interactivity for features such as form validation, dynamic cart updates, and polling submissions.
  • Bootstrap: Utilized for a responsive layout and pre-designed components, ensuring a mobile-friendly interface.

Backend Technologies

  • Node.js: Used as the JavaScript runtime for server-side execution, handling requests and responses efficiently.
  • Express.js: A web framework for Node.js, simplifying routing and middleware handling for organized server code.
  • File System Module: Used to read from and write data to formdata.json, ensuring persistent storage of user submissions.

Data Storage

JSON: A lightweight data format used for storing form submissions in a structured manner, facilitating easy manipulation and retrieval of data.
